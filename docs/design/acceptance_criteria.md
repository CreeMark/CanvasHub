# 验收标准 (Acceptance Criteria)

## 1. 性能指标
| 模块 | 指标 | 说明 |
| :--- | :--- | :--- |
| **启动时间** | < 2秒 | 冷启动到登录界面 |
| **登录响应** | < 500ms | 数据库查询与Token生成 |
| **同步延迟** | < 100ms | 局域网环境下，A画笔->B显示 |
| **并发用户** | > 10人/房间 | 单个房间支持的同时在线绘画人数 |
| **内存占用** | < 200MB | 正常绘画状态下的内存峰值 |
| **Undo/Redo**| > 50步 | 支持的撤销步数历史 |

## 2. 功能验收
### 2.1 用户系统
- [ ] 注册：支持用户名/密码，密码需Hash存储 (SHA256)。
- [ ] 登录：验证成功后返回Session/Token。
- [ ] 状态：显示当前在线用户列表。

### 2.2 项目管理
- [ ] 创建：新建空白画布，可自定义名称。
- [ ] 列表：显示所有可加入的项目/房间。
- [ ] 删除：仅创建者可删除项目。

### 2.3 绘画功能
- [ ] 画笔：支持颜色、粗细调节。
- [ ] 橡皮擦：支持粗细调节。
- [ ] 撤销/重做：快捷键 (Ctrl+Z/Y) 及按钮支持。
- [ ] 清空：一键清空画布。

### 2.4 协作功能
- [ ] 实时同步：笔迹在松开鼠标或每隔N个点时发送。
- [ ] 聊天：文本消息实时广播，显示发送者ID。

### 2.5 数据存储
- [ ] 数据库：MySQL 8.x C API 存储用户信息与项目元数据。
- [ ] 文件保存：支持保存为自定义格式 (.canvas) 或导出 PNG。

## 3. 质量控制
- [ ] **内存泄漏**：Valgrind 检测无泄漏 (严格管理 malloc/free)。
- [ ] **代码规范**：符合 Linux Kernel 风格或 MISRA-C。
- [ ] **单元测试**：使用 Unity/CMocka，核心逻辑覆盖率 > 90%。
